## 二、命令详解 × 场合矩阵

### 2.1 `xw` — 上下文心跳

**是什么：** 一键查看当前会话的 Token 使用率、任务类型、预警阈值。相当于系统的「仪表盘」。

**什么时候用：**

| 场合 | 说明 |
|------|------|
| 🔋 长对话中段 | 对话超过 30 轮后，看一眼 Token 消耗，避免上下文溢出 |
| 🔀 切换任务前 | 确认当前被检测到的任务类型是否正确，作为心理锚点 |
| 🚨 感觉回答变慢 | 检查是否接近 Token 预警线，决定是否需要 `/new` 开新会话 |
| 🧘 每天启动时 | 开终端后第一件事 `xw`，像早上看仪表盘一样确认系统就绪 |

**典型工作流：**
```
$ xw
  ━━━ Token 监控 ━━━
  已用: 12,340 / 200,000 (6.2%)
  当前任务: 💻 编程
  预警阈值: 80%
```

---

### 2.2 `xw flow` — 思维轨迹回放

**是什么：** 展示当前会话中任务的跳转路径——从「修 Bug」到「写文档」再到「重构代码」的完整流转图。

**什么时候用：**

| 场合 | 说明 |
|------|------|
| 🔍 迷失方向时 | 感觉干了半天不知道在做什么，`xw flow` 帮你拉回主线 |
| 📝 写日报/周报 | 用流转图快速回忆今天做了什么、任务怎么衔接的 |
| 🔀 频繁被打断 | 被各种小事打断后，看一眼流转图重建心理上下文 |
| 🤝 交接/同步 | 把流转图贴给协作者，快速同步当前进度 |
| 🎯 判断是否偏航 | 对比最初的意图和实际的流转路径，识别隐性偏航 |

**典型输出：**
```
📊 任务流转追踪 (xw flow)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  09:15 💻 编程 → 修复登录Bug
  11:30 📄 文档 → 写API变更记录
  13:00 💻 编程 → 重构auth模块
  15:45 📝 内容 → 写技术方案评审
  ─────────────────────────
  流转次数: 3 | 主要任务: 💻 编程 (50%)
```

---

### 2.3 `xw report` — 行为镜像复盘

**是什么：** 一键生成个人行为回顾报告，基于 `analytics.db` 中的画像数据。这是系统对你的「客观镜像」——不带评价，只呈现数据。

**什么时候用：**

| 场合 | 说明 |
|------|------|
| 🌙 每天结束前 | 生成当日行为报告，做一次 5 分钟自我复盘 |
| 📅 每周回顾 | 周五下午执行，看看一周的认知模式变化 |
| 📈 月度趋势 | 拉长周期观察自己的任务偏好、高效时段、认知偏好漂移 |
| 🎯 设定目标前 | 先看 `xw report` 了解现状，再制定改进目标 |
| 🔄 换项目时 | 回顾上一阶段的行为特征，为新项目做好心理过渡 |

---

### 2.4 `xw idea` — 零摩擦灵感捕获 ★ 核心命令

**是什么：** 灵感白板层的唯一入口。三字指令：`catch`（捕获）、`list`（浏览）、`flush`（缝合）。

#### 2.4.1 `xw idea "..."` — 瞬时捕获 ⌃I

**什么时候用：**

| 触发信号 | 示例 |
|----------|------|
| 💡 编码时闪过架构优化想法 | `xw idea "把认证层从中间件抽成独立service"` |
| 🐛 发现一个疑似 Bug 但没时间验证 | `xw idea "怀疑缓存失效逻辑在并发下有race condition"` |
| 📖 读到一句醍醐灌顶的话 | `xw idea "Kernel code should be boring — Linus"` |
| 🔧 想到一个工具改进点 | `xw idea "xw flow 应该支持按时间段过滤"` |
| 🤔 对当前方案产生怀疑 | `xw idea "当前微服务拆分粒度是不是太细了？"` |
| 💬 想对未来的自己说句话 | `xw idea "下次重构先写集成测试，别再裸奔了"` |

**核心原则：** 想法的「捕获」和「处理」必须解耦。捕获发生在想法产生的瞬间（<50ms），处理推迟到 flush 阶段（可以几小时甚至几天后）。**绝对不要在编程流中停下来去想这个想法好不好、值不值得记——先记下来。**

#### 2.4.2 `xw idea list` — 浏览碎片 ⌃L

**什么时候用：**

| 场合 | 说明 |
|------|------|
| ☕ 休息间隙 | 泡咖啡时扫一眼白板，看看今天积累了什么碎片 |
| 🔀 切换任务前 | 确认有没有与此相关的历史碎片可以一起处理 |
| 📊 flush 之前 | 先 list 看一眼碎片数量和主题，决定是否现在 flush |
| 🧹 清理心理缓存 | 知道想法已经安全落盘了，安心继续写代码 |

#### 2.4.3 `xw idea flush` — 语义缝合 ⌃F

**是什么：** 把白板中所有 pending 碎片提取出来，结合 `xw flow` 任务流转轨迹，交给 LLM 进行语义层面的归纳、关联、重组，最终生成一份结构化的产品思考报告。

**什么时候用：**

| 触发时机 | 说明 |
|----------|------|
| 🕐 每天一次 | 下班前或睡前 flush 一次，相当于「日结」 |
| 📦 功能开发完成 | 一个 feature 做完后 flush，把过程中的碎片想法沉淀为后续路线图 |
| 🧠 感觉脑子很乱 | 当想法碎片累积到「不整理就焦虑」的程度，立刻 flush |
| 🗓️ 每周深度复盘 | 周末 flush 一周的积累，产出一份高质量的周度思考报告 |
| 🚀 启动新项目前 | flush 历史碎片，看看有没有之前想到但没执行的好点子 |

**flush 产出：**
- 落盘到 `/Volumes/13384923891/hermes-agent/context-system/Whiteboard/Whiteboard_Log_YYYYMMDD.md`
- 可用 Obsidian 直接挂载 `Whiteboard/` 目录进行深度阅读和二次编辑
- 每个被缝合的碎片状态从 `pending` 变为 `flushed`

## 三、典型日工作流

```
  09:00  $ xw                       ← 看一眼仪表盘
  09:15  开始编码...
  10:30  $ xw idea "重构user模块"   ← ⌃I 闪念捕获
  11:00  $ xw idea "API版本管理方案" ← ⌃I 又闪了一下
  12:00  $ xw flow                  ← 看看上午的流转
  12:05  $ xw idea list             ← ⌃L 瞄一眼碎片
  14:00  继续编码...
  15:30  $ xw idea "xw idea应该支持标签" ← ⌃I
  16:00  $ xw                       ← 检查Token余量
  18:00  $ xw flow                  ← 回顾全天流转
  18:05  $ xw idea flush            ← ⌃F 日结缝合
  18:10  $ xw report                ← 生成行为报告
  18:15  🏠 下班
```

## 四、反模式（不要这样做）

| 反模式 | 为什么不对 | 正确做法 |
|--------|-----------|----------|
| ❌ 捕获灵感后立刻停下来思考 | 打断编程流，丢失心流状态 | ✅ 记完就走，flush 时再深度处理 |
| ❌ 从不 flush，碎片堆积如山 | 碎片失去了关联价值，变成噪音 | ✅ 至少每天 flush 一次 |
| ❌ 在 flush 前手动分类整理 | 分类应该留给 LLM 语义缝合 | ✅ 直接 flush，让系统做关联 |
| ❌ 用 xw idea 代替 TODO list | 灵感 ≠ 待办事项，混在一起会污染 | ✅ TODO 用专门工具，xw idea 只管灵感和产品思维 |
| ❌ 每个小想法都开新会话讨论 | 浪费 Token，打散思维连续性 | ✅ 存到 xw idea，攒一批再集中讨论 |

## 五、⌃I 悬浮便签：配置你的终端快捷键

`xw idea popup` 会打开一个独立的浮动便签窗口（tkinter GUI），⌘Enter 提交，Esc 关闭。底层直连 `whiteboard.py` → `analytics.db`，零网络请求。

### 5.1 Terminal.app（你当前使用的终端）

```
步骤 1: Terminal → Settings → Profiles → 选中当前 Profile → Keyboard

步骤 2: 绑定 ⌃I → 弹出便签
  点 [+] 按钮添加：
    Key:        按 ⌃ + I 组合键
    Modifier:   ^ Control
    Action:     Send Text
    文本框:     xw idea popup\n

步骤 3: 绑定 ⌃L → 浏览碎片
  点 [+] 按钮添加：
    Key:        按 ⌃ + L 组合键
    Modifier:   ^ Control
    Action:     Send Text
    文本框:     xw idea list\n

步骤 4: 绑定 ⌃F → 缝合报告
  点 [+] 按钮添加：
    Key:        按 ⌃ + F 组合键
    Modifier:   ^ Control
    Action:     Send Text
    文本框:     xw idea flush\n

步骤 5: 删除 ⌃F 原有绑定
  ⚠️ Terminal.app 的 ⌃F 默认是「向前移动字符」
  在 Keyboard 列表中找到原有的 ⌃F 条目，选中后点 [-] 删除
```

配置完成后，关闭设置窗口。新开一个终端窗口（或 `source ~/.zshrc`），然后按 `⌃I` 试试——应该弹出一个深色悬浮便签窗口。
