## 五维度数据模型

```
                    ┌─────────────────┐
                    │   user_prefs    │ ← 你喜欢什么
                    │ (显式偏好记录)    │
                    └────────┬────────┘
                             │
    ┌──────────┐    ┌────────▼────────┐    ┌──────────┐
    │  ideas   │    │  session_tasks  │    │  seeds   │
    │ (闪念)   │───→│ (行为+元目标)    │←───│ (深思)   │
    │ source:  │    │ event_note:     │    │ domain:  │
    │ popup/   │    │ 为什么做这个     │    │ 产品设计  │
    │ cli/     │    │ related_task:   │    │ 系统架构  │
    │ reflect  │    │ 灵感≠任务时留空   │    │ 方法论    │
    └──────────┘    └─────────────────┘    └──────────┘
```

## 画像维度定义

### 维度 1：行为模式（做什么）
来源：`session_tasks.description` + `session_tasks.task_type`
回答：用户的时间花在哪些事上？任务之间如何跳转？

### 维度 2：意图模式（为什么做）
来源：`session_tasks.event_note`
回答：用户做这些事的深层目的是什么？元目标是否在重复？

### 维度 3：思维特征（怎么想）
来源：`ideas.raw_idea`（闪念）+ `seeds.content`（深思）
回答：用户在什么情况下产生想法？想法是任务附庸还是独立洞察？

### 维度 4：偏好倾向（喜欢什么）
来源：`user_preferences` + `ideas.source` 分布
回答：用户偏好自动化还是手动？行动导向还是思考导向？

### 维度 5：表达风格（怎么说）
来源：语言镜像层分析（Mirror 报告）
回答：用户如何表达需求？使用什么隐喻体系？

## 画像进展报告（每 50 次会话触发）

当 `SELECT COUNT(DISTINCT session_id) FROM session_tasks` 达到 50 时，自动生成：

1. **行为趋势** — 任务类型分布变化、高效时段
2. **意图演化** — 元目标是否在聚焦？新关注领域？
3. **思维密度** — 灵感/种子的产生频率、深思 vs 闪念比
4. **偏好收敛** — 哪些偏好从低置信度变为高置信度
5. **镜像更新** — 与上次 Mirror 报告的差异

## 与决策参谋 (xw-advisor) 的关系

```
数字代理层 = 数据基础设施 + 长期画像
决策参谋   = 基于画像的实时建议引擎

xw-advisor 当前能力: 偏好记录 + on/off 开关
xw-advisor 目标能力: 当用户讨论产品设计时，自动从 seeds 中提取相关想法
                      → 给出「你之前想过...，和现在的...结合可以...」
```
