## 一、核心愿景

灵感白板不是一个「待办清单」，不是一个「笔记应用」，它是一个**思维碎片的即时着陆场**。

用户脑子里随时会冒出零碎想法——「这个功能的交互能不能改成滑动？」「那篇文章的标题起得不好」「今天的会议让我想到一个新的产品方向」。这些念头像飘在空中的蒲公英，伸手抓不住就飞走了。

白板要做的是：一个张开口的布袋。用户只需一个动作，布袋张开，蒲公英丢进去，存完就走，不问任何问题。

攒够了，AI 自动帮你把蒲公英分类整理成可读的报告。

## 二、核心设计原则（向用户反复确认过的 8 条铁律）

### 原则 1：零摩擦捕获 — 50ms 硬约束

灵感是脆弱的。从「脑子里闪过一个念头」到「记录完成」如果超过 3 秒，这个念头大概率消失。

**硬性要求：**
- 按下全局热键 → 窗口完全渲染 < 200ms
- 输入文字 → 提交 → 数据写入 SQLite < 50ms
- **捕获期间绝对禁止发起任何网络请求**（包括 LLM 调用）
- 提交后窗口自动消失，不占用户一秒注意力

### 原则 2：真正的全局可达 — 跳出终端

第一次做的终端内快捷键绑定是错的。用户在 Safari 里看网页、在 VS Code 里写代码、在飞书里开会时，根本不在终端里。

**要求：**
- 全局热键在 macOS 系统任意位置都能唤起
- 实现方式：Carbon Event Manager 级别的系统全局热键注册
- **不是 Terminal.app 的快捷键绑定，不是 iTerm2 的 Key Binding**

### 原则 3：自包含 — 用户拒绝改系统设置

用户明确拒绝手动配置终端快捷键、拒绝改系统偏好设置。

**所有配置必须内建在工具自己的设置面板里。** 全局热键、提交键、透明度——全部在便签内部的 ⌘K 面板中可调节，设置即时生效，无需手动重启任何东西。

### 原则 4：色彩是识别信号，不是装饰

v1.0 用了深色主题——融入终端背景，毫不起眼。

**用户要求的视觉规格（逐像素确认过）：**

```
┌─────────────────────────────────────┐
│ 🪐 灵感捕获              半透明 88%  │ ← 标题栏 #FFD700（金黄色）
├─────────────────────────────────────┤
│                                     │
│   写下此刻的想法……                   │ ← 内容区 #FEF9EF（暖奶油色）
│                                     │
│                                     │
│                                     │
├─────────────────────────────────────┤
│              [ 放弃 ]   [ ⇧↵ 捕获 ] │ ← 按钮区，提交键 #FF6B35（亮橙色）
└─────────────────────────────────────┘
```

关键视觉属性：
- 始终置顶（浮在任何窗口上方，包括全屏应用）
- 半透明（默认 88%，可调 25%~100%）
- 圆角，无厚重标题栏边框
- 不像终端窗口，不像系统对话框，一眼就知道「这是灵感便签」
- 亮色设计确保在任何背景（暗色终端 / 白色网页 / 深色 IDE）上都能一眼辨认

### 原则 5：提交键偏好 — Shift+Enter

这是用户反复确认过的。不是 ⌘Enter，不是单纯 Enter。

**Shift+Enter 提交。** Esc 关闭/丢弃。⌘K 打开设置。

### 原则 6：捕获时零决策负担

捕获时不问任何问题。没有分类选择、没有标签输入、没有标题栏、没有「这个重要吗」的确认框。

只有一个文本框。打完字 → 提交 → 完。

后台自动携带的隐形标签（用户不需要知道）：
- 时间戳（精确到秒）
- 当前会话 ID
- 当前任务类型（💻编程 / 📝内容 / 📄文档 / 📋通用）
- 状态（pending → flushed）

### 原则 7：捕获与处理彻底解耦（核心工作流）

```
捕获（< 50ms，纯本地）     →    处理（按需触发，可延迟几小时甚至几天）
  xw-idea "..."                xw-idea flush
  存入 analytics.db              AI 读取所有 pending 碎片
  (pending 状态)                 语义分类 → 发现关联 → 生成报告
                                落盘 Whiteboard/Whiteboard_Log_YYYYMMDD.md
                                标记为 flushed
```

**用户永远不手动分类。** 分类留给 AI 在 flush 时做。

flush 触发时机：
- 每天一次（下班前或睡前）
- 功能开发完成后
- 碎片累积到「不整理就焦虑」时
- 周末深度复盘

### 原则 8：数据跟着人走，不跟着机器走

- 所有数据存在外挂盘 `/Volumes/13384923891/hermes-agent/`
- 不在 iCloud、不在 Dropbox、不在任何云端
- 换一台 Mac，插上盘，所有灵感都在
- 备份 = 拷贝整个盘
- Whiteboard/ 目录可直接被 Obsidian 挂载为知识库

## 四、功能需求规格

### 4.1 捕获入口（三个入口，一个主力）

| 入口 | 优先级 | 占比 | 触发方式 |
|------|--------|------|---------|
| 全局热键唤起 | P0 必须 | 90% | 任何 App 中按 ⌃I（可配置） |
| 终端命令 | P1 必须 | 9% | `xw-idea "直接写灵感"` |
| 桌面 App 双击 | P2 建议 | 1% | `~/Desktop/灵感便签.app`（AppleScript 编译，纯备选） |

### 4.2 P0 必须实现

1. **全局热键守护进程 (xw-hotkey)**
   - macOS Carbon Event Manager 注册全局热键
   - 配置文件：`context-system/popup_config.json`
   - JSON 格式：`{"hotkey": {"keycode": 34, "modifiers": 4096}, "opacity": 0.88, "submit_key": "Shift-Return", "cancel_key": "Escape"}`
   - 支持热重载（设置面板改热键 → 守护进程自动重启加载新配置）
   - 开机自启（LaunchAgent）

2. **悬浮便签窗口 (xw-idea-popup)**
   - macOS 原生 GUI（tkinter 实现，零外部依赖）
   - 视觉规格严格按原则 4
   - 始终置顶 + 半透明（默认 88%）
   - 窗口位置：屏幕中上部，200px 高，500px 宽
   - 提交动画：绿色 ✓ 勾 + 自动关闭（800ms 后）

3. **设置面板 (⌘K)**
   - 全局热键录制器：点击 → 按下组合键 → 即时保存
   - 提交快捷键录制器
   - 取消快捷键录制器
   - 透明度滑块（25%~100%）
   - 恢复默认按钮
   - 所有设置持久化到 `popup_config.json`

4. **数据写入**
   - 底层直连 `whiteboard.py` → `analytics.db`
   - 调用方式：`python3` 执行 Python 脚本调用 `capture_idea()`
   - **不直接操作数据库**，不绕过 whiteboard.py

### 4.3 P1 必须实现

5. **终端 CLI (xw-idea)**
   - `xw-idea "文字"` → 捕获
   - `xw-idea list` → 列出 pending
   - `xw-idea flush` → AI 缝合（唯一触发 LLM 的操作）
   - 所有操作 < 100ms（flush 除外）

6. **灵感列表 (xw-idea list)**
   - 显示 pending 碎片的 ID、时间、任务类型、内容摘要
   - 不触发网络请求

### 4.4 P2 建议实现

7. **桌面 App 备选入口**
   - AppleScript 编译的 `.app`，双击弹出原生 macOS 对话框
   - 单行输入，功能完整但交互简陋
   - 纯备选，主力还是全局热键

8. **菜单栏小图标**（未来）

## 五、数据架构

### 5.1 数据库表

表名：`whiteboard_ideas`（在 `analytics.db` 中）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增 |
| timestamp | TEXT | 精确到秒，格式 "YYYY-MM-DD HH:MM:SS" |
| session_id | TEXT | 当前会话 ID（如 "STANDALONE" 或 Hermes session ID） |
| current_task_type | TEXT | 任务类型：💻编程 / 📝内容 / 📄文档 / 📋通用 |
| raw_idea | TEXT | 用户输入的原始文字 |
| status | TEXT | 'pending'（未缝合）/ 'flushed'（已缝合） |

### 5.2 数据流

```
用户输入 → 便签窗口 / CLI → whiteboard.py::capture_idea() → analytics.db (whiteboard_ideas, status=pending)
                                                                           ↓
                                     xw-idea flush → AI 语义缝合 → Whiteboard/Whiteboard_Log_YYYYMMDD.md
                                                                           ↓
                                                              UPDATE status='flushed'
```

### 5.3 不可触碰的边界

- ❌ 不要直接读写 analytics.db（只通过 whiteboard.py / db.py）
- ❌ 不要修改 whiteboard.py（它是数据层的唯一写入入口）
- ❌ 不要修改 Personal_OS_Manifest.md（系统宪法）

## 九、验收标准

1. 在任何 App（Safari / VS Code / 飞书 / 桌面）中按 ⌃I，200ms 内弹出悬浮便签
2. 便签外观符合第六节的逐像素规格
3. 输入文字 → Shift+Enter → 窗口显示 ✓ → 自动关闭 < 800ms

5. ⌘K 面板能修改全局热键、提交键、透明度，改完即时生效
6. 修改后的快捷键在设置面板关闭后立即生效（守护进程自动重载）
7. `xw-idea flush` 能把 pending 碎片缝合为 Markdown 报告，存到 Whiteboard/ 目录
8. 所有数据在 `/Volumes/13384923891/hermes-agent/` 外挂盘中
9. 捕获和 list 操作不发起任何网络请求

## 十、已知问题（来自用户反馈）

> 用户说：「当前的 whiteboard 工具的设计完全适应不了」

可能的问题方向（需要排查）：
- 全局热键不稳定？守护进程容易挂？
- 半透明/置顶效果不符合预期？
- 窗口在特定 App（全屏应用、Stage Manager）中无法显示？
- 提交的灵感无法在 flush 时正确关联？
- 窗口位置/大小不符合使用习惯？
- Shift+Enter 在某些键盘布局中失效？

## 十一、v2.5 升级：上下文感知缝合

flush 现分两阶段：
1. 格式输出—照常生成 Obsidian 格式报告
2. 深度缝合—prepare_flush_context() 关联 session_tasks 中的任务上下文和 event_note 元目标，生成 .flush_prompt.md 供 LLM 分析
关联窗口：每条灵感前后 ±60 分钟内的任务。元目标来自 session_tasks.event_note（空则用 description 替代）。
