## 1. 产品概述

### 1.1 产品定位

**灵感便签 (CtxIdea)** 是 Personal OS 五层架构中「02_灵感白板层」的 GUI 前端。它是一条「思维碎片的即时着陆跑道」——脑子里闪过任何念头，按下快捷键，0.3 秒内弹出输入窗口，打完字提交即存入外挂盘数据库，窗口自动消失。

**一句话定义：** 一个张开嘴的布袋，你不必分类、不必命名、不必审视——把念头丢进去，它就帮你收好了。等攒够了，AI 帮你把零散念头串成结构化的思路。

### 1.2 核心痛点

- 灵感/杂念转瞬即逝，3 秒内不记录就会消失
- 现有的记录工具（备忘录、Notion、Obsidian）打开路径太长：解锁手机 → 找 App → 新建笔记 → 写标题 → 写内容 → 保存。这个过程中灵感已经蒸发。
- 终端用户不想离开键盘去操作鼠标
- 用户拒绝在系统设置里手动绑定快捷键——一切配置必须内建在工具自己的设置面板中

### 1.3 用户画像

- macOS 用户，外挂硬盘使用者
- 非技术背景（不写代码，不理解技术细节）
- 偏好：自包含工具、亮眼的视觉设计、始终置顶半透明悬浮窗、Shift+Enter 提交
- 核心使用场景：编程/写作/阅读/开会途中，脑子里闪过任何想法时，立即捕获

## 2. 功能规格

### 2.1 核心功能清单

#### F1: 全局热键唤起

```
优先级: P0 (必须有)
```

- **默认热键:** ⌃I（Control + I）
- **作用范围:** 系统全局——在任何 App 中按下均可唤起
- **唤起速度:** 窗口出现 < 200ms
- **实现方式:** 应用启动后注册 macOS 全局快捷键（通过 Carbon Event Manager 或 CGEvent API）。热键触发时，如果窗口已存在则聚焦并清空内容，如果不存在则创建新窗口。

## 3. 技术架构要求

### 3.1 推荐技术栈

以下是我的建议——你有最终决策权，选你最擅长的方式：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Swift + SwiftUI** (推荐) | macOS 原生体验最佳，窗口管理完善，系统级热键 API 最稳定 | 与 Python 后端的集成需要进程间通信 |
| **Electron + Node.js** | 你（Claude）擅长 JavaScript/HTML/CSS，开发速度快 | 体积大，内存占用高，非原生体验 |
| **Python + PyObjC** | 可直接 import whiteboard.py，零 IPC 开销 | macOS 原生体验受限，全局热键需 Carbon 桥接 |
| **Swift + PythonKit** | 原生 GUI + 直接调用 Python 模块 | PythonKit 兼容性不稳定 |

**我的推荐：Swift + SwiftUI**  
理由：这个工具的核心价值在于「顺滑的 macOS 原生体验」——全局热键零延迟、窗口动画流畅、系统级集成完美。用 Swift 实现 GUI 层，用 Process 或嵌入式 Python 调用 whiteboard.py 完成数据写入。

**与 Python 后端的集成方式（如果你选 Swift）：**
1. 应用启动时 spawn 一个 Python 子进程运行一个微型 HTTP server（Flask/FastAPI），监听 `localhost:19876`
2. 捕获灵感时 → 发送 POST `http://localhost:19876/capture` with `{"text": "..."}`
3. Python server 调用 `capture_idea()` 并返回 `{"id": 123}`
4. 应用退出时 kill 子进程

或者更简单的方式：
```swift
// 直接调用系统 Python3 执行一行脚本
let process = Process()
process.executableURL = URL(fileURLWithPath: "/usr/bin/python3")
process.arguments = ["-c", """
import sys; sys.path.insert(0, '/Volumes/13384923891/hermes-agent/context-system')
from core.whiteboard import capture_idea
print(capture_idea('STANDALONE', '📋 通用', '\(text)'))
"""]
```

**Python 子进程方案的注意事项：**
- 每次捕获 spawn 一个 Python 进程的开销约 50-100ms，在可接受范围内
- 如需更低延迟，使用长生命周期的 Python HTTP server
- 确保外挂硬盘路径在 Python 子进程中可访问

### 3.2 全局热键实现

**macOS 全局热键核心 API：**
- `RegisterEventHotKey` (Carbon) — 最可靠，老的但稳定
- `NSEvent.addGlobalMonitorForEvents(matching:)` — 较新，但需要辅助功能权限
- `CGEvent.tapCreate` — 最低层级，最灵活

**推荐：** `RegisterEventHotKey` + `NSEvent.addGlobalMonitorForEvents` 双保险。

**必须处理：**
- 首次运行时请求「辅助功能」权限（系统设置 → 隐私与安全性 → 辅助功能）
- 权限未授予时显示引导提示窗口
- 热键修改后即时注销旧注册、注册新热键

## 4. 交互设计细节

### 4.1 键盘快捷键全集

| 快捷键 | 作用 | 作用范围 |
|--------|------|----------|
| ⌃I | 唤起/聚焦便签窗口 | 系统全局 |
| ⇧↵ (Shift+Enter) | 提交灵感内容 | 便签窗口内 |
| Esc | 关闭便签窗口（丢弃内容） | 便签窗口内 |
| ⌘W | 关闭便签窗口（同 Esc） | 便签窗口内 |
| ⌘K | 打开设置面板 | 便签窗口内 |
| ⌘= | 增加透明度 5% | 便签窗口内 |
| ⌘- | 减少透明度 5% | 便签窗口内 |
| ⌘Q | 退出应用 | 便签窗口内 |

### 4.2 窗口生命周期

```
[应用启动] → 注册全局热键 → 等待
                              ↓
                     用户按 ⌃I → 窗口不存在? → 创建窗口 → 显示 + 聚焦
                              → 窗口已存在? → 聚焦 + 清空输入区 + 提到最前
                              ↓
                     用户输入文字...
                              ↓
                     按 ⇧↵ → 调用 capture_idea() → 显示 ✓ → 800ms后窗口隐藏
                              ↓
                     按 Esc → 窗口隐藏（不退出应用，守护在后台）
                              ↓
[应用退出] → 注销全局热键 → 保存窗口位置和大小 → 退出
```

**关键行为：**
- 「关闭窗口」≠「退出应用」。应用在后台持续运行以监听全局热键。
- 窗口隐藏（非销毁）以保持快速唤起的性能。
- 点击 Dock 图标 → 如果窗口隐藏则显示，如果窗口显示则隐藏（toggle）。
- 右键 Dock 图标 → 菜单：显示便签 / 设置 / 退出。

## 5. 产品设计原则（指导你的每一个决策）

这些是构建过程中沉淀的核心设计原则。当你在实现中遇到两难选择时，回到这些原则：

### 原则 1：零摩擦是生存底线

灵感在 3 秒内不记录就会消失。从「念头闪过」到「记录完成」的每一步操作都在杀死灵感。**删掉一切不必要的步骤：** 不要求标题、不要求分类、不要求确认、不弹窗说「保存成功」——直接存，安静地存，存完就走。

### 原则 2：捕获与整理必须解耦

灵感捕获和灵感处理是两件完全不同的事，必须在时间和空间上彻底分离。捕获发生在想法产生的瞬间（50ms 以内），处理发生在几小时甚至几天后的 flush 阶段。**捕获时大脑负责产生想法，flush 时 AI 负责理解想法。**

### 原则 3：自包含——用户不应该离开工具去配置工具

所有设置必须内建在工具自己的设置面板中（⌘K）。用户永远不需要打开系统设置、终端、或编辑 JSON 文件来修改配置。

### 原则 4：色彩不是装饰，是识别信号

这个便签的视觉设计——亮黄色标题、暖奶油色底、橙色按钮——不仅是审美选择，更是功能需求。它必须能在任何背景下（暗色终端、白色网页、黑色 IDE）一眼被识别。半透明 + 始终置顶确保它不被任何窗口完全遮挡。

### 原则 5：Shift+Enter 不是拍脑袋选的

用户经过了反复迭代确认——他们不要 ⌘Enter，就要 Shift+Enter。这是经过验证的个人偏好，不要擅自更改默认值。

### 原则 6：数据跟着人走，不跟着机器走

一切存在外挂硬盘上（`/Volumes/13384923891/hermes-agent/`）。换一台 Mac，插上盘，所有灵感都在。这条原则决定了所有文件路径的设计。

### 原则 7：工具应该自我成长

这个系统不是静态的——每一次使用都在积累数据，每一次 flush 都在产生新的洞察。你的代码应该为这种「进化」留出空间。不要写死任何东西，配置文件用 JSON，数据结构留扩展字段。
